ProfMatch — HTML-only page mockups

Every file is self-contained semantic HTML with neutral grayscale styling and dummy data — structure and content only, no brand design, no JavaScript. Feed any file into the redesign workflow on its own. The full feature inventory is in docs/FEATURES.md.

Public pages

  • landing.html — / hero search, comparison, how-it-works, FAQ
  • discover.html — /professors search, filters, result cards
  • professor-detail.html — /professors/[id] profile, summary, papers
  • universities.html — /universities + university + department pages
  • pricing.html — /pricing
  • auth.html — /signin + /signup

Signed-in pages

  • dashboard.html — /dashboard home overview
  • match.html — /match thresholds + ranked results (and /intake wizard)
  • workflow.html — /saved, /compare, /board
  • profile.html — /profile academic profile + /recommend department request

Admin pages

  • admin.html — /admin dashboard, /admin/scans, /admin/onboarding

Shared components

  • components.html — modals (login, report, outreach draft, confirm), toasts, chips, cards, nav shells
